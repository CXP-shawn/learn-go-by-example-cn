## 🤔 先想一个问题

假设你想创建一个计数器，每次调用它就返回下一个整数：第一次调用返回 1，第二次返回 2，以此类推。

在其他语言里你可能会用一个全局变量来记录当前计数。但全局变量有很多问题：难以管理状态、线程不安全、容易被误修改……

Go 的解决方案是**闭包**——让函数自己"记住"一个私有的状态变量，每次调用都能访问和修改这个变量，而外部代码完全无法直接访问它。这就是闭包的魔力。

---

## 📖 看代码

```go
// Go 支持匿名函数，并能用其构造闭包。
// 匿名函数在你想定义一个不需要命名的内联函数时是很实用的。

package main

import "fmt"

// `intSeq` 函数返回一个在其函数体内定义的匿名函数。
// 返回的函数使用闭包的方式隐藏变量 `i`。
// 返回的函数隐藏变量 `i` 以形成闭包。
func intSeq() func() int {
	i := 0
	return func() int {
		i++
		return i
	}
}

func main() {

	// 我们调用 `intSeq` 函数，将返回值（一个函数）赋给 `nextInt`。
	// 这个函数的值包含了自己的值 `i`，这样在每次调用 `nextInt` 时，都会更新 `i` 的值。
	nextInt := intSeq()

	// 通过多次调用 `nextInt` 来看看闭包的效果。
	fmt.Println(nextInt())
	fmt.Println(nextInt())
	fmt.Println(nextInt())

	// 为了确认这个状态对于这个特定的函数是唯一的，我们重新创建并测试一下。
	newInts := intSeq()
	fmt.Println(newInts())
}
```

## 🔍 师兄给你逐行拆

闭包是函数式编程里一个非常重要的概念，理解它需要一点耐心。师兄带你一步步拆解。

### 什么是匿名函数

先说"匿名函数"。在 Go 里，函数不一定要有名字。没有名字的函数就叫**匿名函数**（Anonymous Function）。

通常我们定义函数是这样的：
```go
func myFunc() int {
    return 42
}
```

匿名函数省去了函数名：
```go
func() int {
    return 42
}
```

匿名函数可以直接调用（立即执行函数），也可以赋给变量，或者作为返回值。

### 理解 intSeq 的函数签名

```go
func intSeq() func() int {
```

这行代码一眼看上去有点复杂，我们来仔细拆：

- `func intSeq()`：定义一个名叫 `intSeq` 的函数，没有参数
- `func() int`：这是 **返回值类型**！它表示 `intSeq` 返回的是一个函数类型——具体来说，是一个"没有参数、返回 int"的函数

在 Go 里，函数类型就像 `int`、`string` 一样，是合法的类型。`func() int` 就是一种函数类型，代表"无参数，返回一个 int 的函数"。

所以 `intSeq` 是一个**返回函数的函数**，也叫"高阶函数"（Higher-order function）。

### intSeq 的函数体——闭包的关键

```go
func intSeq() func() int {
    i := 0
    return func() int {
        i++
        return i
    }
}
```

逐行解析：

**`i := 0`**：在 `intSeq` 的函数体里声明了一个局部变量 `i`，初始值为 0。

**`return func() int { ... }`**：返回一个匿名函数。这个匿名函数的函数体里用到了外层函数的变量 `i`。

**这里就是闭包的魔法！**

正常情况下，当一个函数执行完毕，它的局部变量就会被销毁。但这里的 `i` 是个例外——因为内层的匿名函数引用了它，Go 会让 `i` 继续存活在堆内存上，并且绑定给返回的那个匿名函数。这种"函数+它捕获的变量"的组合，就叫**闭包**（Closure）。

形象地说：返回的匿名函数就像一个带着自己小背包的旅行者，背包里装着 `i`。每次这个函数被调用，它就打开背包，把 `i` 加一，再把新值放回去。

### main 函数——闭包的使用

```go
nextInt := intSeq()
```

调用 `intSeq()`，它返回那个携带着 `i=0` 的匿名函数，并把这个函数赋给 `nextInt`。现在 `nextInt` 本身就是一个函数，可以直接调用。

```go
fmt.Println(nextInt())  // 输出 1
fmt.Println(nextInt())  // 输出 2
fmt.Println(nextInt())  // 输出 3
```

每次调用 `nextInt()`：
1. 进入匿名函数体
2. 执行 `i++`（i 从 0 变 1，再到 2，再到 3）
3. 返回当前的 `i` 值

注意！每次调用都修改的是**同一个** `i`，因为 `nextInt` 始终持有对同一个 `i` 的引用。所以输出依次是 1、2、3。

### 新实例拥有独立的状态

```go
newInts := intSeq()
fmt.Println(newInts())  // 输出 1
```

这里重新调用 `intSeq()`，创建了一个**全新的闭包**。这个新的闭包有自己的 `i`，初始值是 0，与 `nextInt` 的 `i` 完全独立。

所以 `newInts()` 输出的是 1，而不是 4。这证明了**不同的闭包实例之间状态是隔离的**，互不影响。

这个特性非常有用：你可以用同一个工厂函数创建多个独立的计数器，它们各自维护自己的状态。

### 闭包的本质

闭包 = 函数 + 它所捕获的外部变量的引用。

要形成闭包，需要满足：
1. 函数内部定义了另一个函数（内层函数）
2. 内层函数引用了外层函数的局部变量
3. 内层函数被返回或传递到外部使用

闭包让函数拥有了"记忆"，每个闭包实例都封装了自己的私有状态，这使得闭包成为实现封装、工厂模式、回调函数等场景的强大工具。

### 闭包在实际开发中的应用

闭包不只是玩具，它在真实项目里随处可见：

**1. 计数器/生成器**（就像本例）

**2. 延迟执行和回调**：
```go
// 在 HTTP 路由里，handler 函数常常是闭包
http.HandleFunc("/", func(w http.ResponseWriter, r *http.Request) {
    fmt.Fprintln(w, "Hello, World!")
})
```

**3. 中间件模式**：外层函数接收配置参数，返回实际的处理函数

**4. goroutine 中传递数据**：
```go
for _, item := range items {
    item := item  // 重新声明，避免闭包捕获循环变量的陷阱
    go func() {
        process(item)
    }()
}
```

## 💡 师兄的碎碎念

**闭包的常见陷阱：循环变量捕获**

在 Go 中使用闭包时，有一个非常经典的坑：

```go
// 错误示例
funcs := make([]func(), 3)
for i := 0; i < 3; i++ {
    funcs[i] = func() { fmt.Println(i) }
}
for _, f := range funcs {
    f()  // 都会打印 3，而不是 0, 1, 2
}
```

为什么？因为所有闭包捕获的是同一个变量 `i` 的**引用**，等到闭包被调用时，循环已经结束，`i` 的值已经是 3 了。

解决方法：在循环内用新变量"固定"当前的值：

```go
for i := 0; i < 3; i++ {
    i := i  // 新的 i 遮蔽了外层 i，每个闭包捕获独立的副本
    funcs[i] = func() { fmt.Println(i) }
}
```
