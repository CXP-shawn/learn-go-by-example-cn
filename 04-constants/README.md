# 第 4 关：Constants（师兄带你学 Go）

## 🎯 这一关你会学到

这一关学的是 Go 的「常量」机制。常量就是程序运行期间永远不会被修改的值，用 `const` 声明。你会看到 Go 的常量比你想象中强大——它支持任意精度运算，而且数值常量没有固定类型，会根据使用场景自动适配，省去很多烦人的类型转换。

package main

import (
 "fmt"
 "math"
)

// `const` 用于声明一个常量。
const s string = "constant"

func main() {
 fmt.Println(s)

 // `const` 语句可以出现在任何 `var` 语句可以出现的地方
 const n = 500000000

 // 常数表达式可以执行任意精度的运算
 const d = 3e20 / n
 fmt.Println(d)

 // 数值型常量没有确定的类型，直到被给定某个类型，比如显式类型转化。

 // 一个数字可以根据上下文的需要（比如变量赋值、函数调用）自动确定类型。

## 🔍 师兄给你逐行拆

### 第一段：包级别的常量声明

```go
const s string = "constant"
```

这行写在 `main` 函数**外面**，属于「包级别常量」，整个文件里的任何函数都能直接用 `s`。注意这里显式写了类型 `string`，就像你点奶茶时特别跟店员说「我要**热**的」，不说也能猜到，但说出来更明确。

常量和变量最本质的区别就一句话：**常量在编译期就确定了，运行期间改不了**。你要是手贱写一句 `s = "other"`，编译器直接报错，绝不手软。

---

### 第二段：函数内部也能声明常量

```go
const n = 500000000
```

这行在 `main` 函数里面，证明 `const` 不是包级别的「特权」，函数内部也能用。这个 `n` 没有显式指定类型，是一个「无类型常量」——先把它理解成一个超级灵活的数字标签，还没贴上「我是 int」或「我是 float64」的标签纸。

---

### 第三段：任意精度的常量运算

```go
const d = 3e20 / n
```

`3e20` 就是 3 × 10²⁰，一个天文数字。这行用常量做除法，算出来的 `d` 是 `6e11`（也就是 600,000,000,000）。

这里有个非常骚的特性：**Go 的常量运算是任意精度的**。普通 `int64` 最大才 9.2 × 10¹⁸，直接塞 `3e20` 早就溢出了。但常量不走普通变量那条路，编译器会用「大数」来做运算，精度理论上没有上限。这就好比你用草稿纸做数学题，想写多少位数字都行，不像计算器有位数限制。

---

### 第四段：显式类型转换——给常量贴标签

```go
fmt.Println(int64(d))
```

`d` 本来是无类型常量，现在用 `int64(d)` 强行给它贴上「我是 64 位整数」的标签。转换之后，小数部分直接截断，输出 `600000000000`。

容易踩的坑：如果 `d` 的值超出了 `int64` 的范围，编译器会直接报错，不是运行时 panic，是**编译期**就挂——这其实是好事，早发现早治疗。

---

### 第五段：上下文自动推断类型——常量的隐身术

```go
fmt.Println(math.Sin(n))
```

`math.Sin` 要求参数是 `float64`。但我们传进去的 `n` 明明是个「无类型常量」，没有明确说自己是 `float64` 啊？

这就是 Go 常量的隐身术：**无类型常量会根据上下文自动变成合适的类型**。`math.Sin` 需要 `float64`，Go 编译器就悄悄把 `n`（500000000）以 `float64` 的身份传进去，你完全不用手写 `float64(n)`。

类比一下：你去不同国家旅游，身上带的是「美元现金」，到哪个国家换汇台就换成当地货币，而不是你提前换好全世界所有国家的货币——常量就是那叠美元现金，用到哪换到哪。

---

### 总结一下容易混淆的点

| 特性 | 变量 `var` | 常量 `const` |
|---|---|---|
| 能否修改 | ✅ 可以 | ❌ 编译报错 |
| 类型是否必须 | 可以推断 | 可以推断（且更灵活）|
| 精度 | 受类型限制 | 编译期任意精度 |
| 类型确定时机 | 声明时 | 使用时（无类型常量）|

记住：常量不是「只读变量」，它是一个完全不同层次的存在，活在编译期，精度更高，用起来更灵活。
