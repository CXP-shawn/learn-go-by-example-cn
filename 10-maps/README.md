# 第 10 关：Map（师兄带你学 Go）

## 🎯 这一关你会学到

这一关你会认识 Go 里最常用的「键值存储」结构——**Map（映射）**，它就像一本通讯录，用名字就能查到电话号码，超级方便。你会学会怎么创建 map、往里存数据、取数据、删数据，以及一个非常实用的「判断键存不存在」的写法。学完之后，你处理各种「查表」类问题会顺手很多。

## 🤔 先想一个问题

你点外卖的时候，平台是怎么根据你的「订单号」找到你那份红烧牛肉饭的？它肯定不是从第一单开始一条条翻，而是直接拿订单号「对号入座」——这背后用的数据结构，就叫 **Map（映射/哈希表）**。

**Map** 是一种「键（Key）→ 值（Value）」的关联数据结构：你给一个唯一的键，它立刻返回对应的值，查找速度极快（平均 O(1)），不管你里面存了 100 条还是 100 万条数据。在其他语言里，它也常被叫做 **哈希（hash）** 或 **字典（dict）**。Go 把它作为内建类型直接支持，不需要额外 import 任何包，拿来就能用。

## 📖 看代码

```go
// _map_ 是 Go 内建的[关联数据类型](http://zh.wikipedia.org/wiki/关联数组)
// （在一些其他的语言中也被称为 _哈希(hash)_ 或者 _字典(dict)_ ）。

package main

import "fmt"

func main() {

 // 要创建一个空 map，需要使用内建函数 `make`：`make(map[key-type]val-type)`。
 m := make(map[string]int)

 // 使用典型的 `name[key] = val` 语法来设置键值对。
 m["k1"] = 7
 m["k2"] = 13

 // 打印 map。例如，使用 `fmt.Println` 打印一个 map，会输出它所有的键值对。
 fmt.Println("map:", m)

 // 使用 `name[key]` 来获取一个键的值。
 v1 := m["k1"]
 fmt.Println("v1: ", v1)

 // 内建函数 `len` 可以返回一个 map 的键值对数量。
 fmt.Println("len:", len(m))

 // 内建函数 `delete` 可以从一个 map 中移除键值对。
 delete(m, "k2")
 fmt.Println("map:", m)

 // 当从一个 map 中取值时，还有可以选择是否接收的第二个返回值，该值表明了 map 中是否存在这个键。
 // 这可以用来消除 `键不存在` 和 `键的值为零值` 产生的歧义，
 // 例如 `0` 和 `""`。这里我们不需要值，所以用 _空白标识符(blank identifier)_ _ 将其忽略。
 _, prs := m["k2"]
 fmt.Println("prs:", prs)

 // 你也可以通过右边的语法在一行代码中声明并初始化一个新的 map。
 n := map[string]int{"foo": 1, "bar": 2}
 fmt.Println("map:", n)
}
```

## 🔍 师兄给你逐行拆

### 第一步：用 `make` 创建一个空 map

```go
m := make(map[string]int)
```

这行代码在干嘛？就是「开一张空白通讯录」。`make` 是 Go 的内建函数，专门用来初始化 map、slice、channel 这类引用类型。`map[string]int` 是类型声明，意思是：键（Key）是 `string` 类型，值（Value）是 `int` 类型。

**为什么一定要用 `make` 而不是直接 `var m map[string]int`？** 这是新手最容易踩的坑！`var m map[string]int` 只是声明了一个 map 变量，它的值是 `nil`，根本没有分配内存空间。你如果直接往 nil map 里写数据，程序会直接 **panic（运行时崩溃）**，报错信息是 `assignment to entry in nil map`。读 nil map 不会崩溃（会返回零值），但写就凉了。所以记住：**要用 map 之前，必须先用 `make` 初始化它**。

类比一下：`var m map[string]int` 就好比你说「我有一个笔记本」，但实际上手里什么都没有；`make(map[string]int)` 才是真的从文具店买了一本空白笔记本回来，现在可以往上写字了。

---

### 第二步：写入键值对

```go
m["k1"] = 7
m["k2"] = 13
```

这两行就是「往通讯录里记两条数据」。语法是 `map名[键] = 值`，非常直观。`"k1"` 是键，`7` 是对应的值。

需要注意的是，map 的键必须是「可比较的（comparable）」类型，也就是可以用 `==` 比较的类型，比如 string、int、bool、指针等。**切片（slice）、map 本身、函数类型不能作为 map 的键**，因为它们不支持 `==` 比较，编译器会直接报错。

另外，**map 里的键是唯一的**。如果你对同一个键赋值两次，后面那次会覆盖前面的，不会报错，也不会有提示，所以要小心。这就像通讯录里同一个名字只能对应一个号码，你改了就是改了。

---

### 第三步：打印整个 map

```go
fmt.Println("map:", m)
```

`fmt.Println` 会把 map 的所有键值对都打印出来，格式大概是 `map[k1:7 k2:13]`。这里有个坑要提前说：**Go 的 map 是无序的**！每次打印的顺序可能不一样，这是 Go 故意设计的（从 Go 1 开始就是随机化的），为了防止开发者依赖 map 的遍历顺序写出有 bug 的代码。如果你需要有序输出，得把键取出来排序后再打印，这个在下一关学 `range` 的时候会碰到。

---

### 第四步：用键取值

```go
v1 := m["k1"]
fmt.Println("v1: ", v1)
```

这就是「查通讯录」：给一个键，拿回对应的值。`m["k1"]` 会返回 `7`，赋值给 `v1`。

**踩坑提示**：如果你查一个不存在的键，Go 不会报错，而是返回该值类型的**零值（zero value）**。`int` 的零值是 `0`，`string` 的零值是 `""`，`bool` 的零值是 `false`。这就带来一个问题：你拿到 `0`，到底是「键存在但值就是 0」，还是「键根本不存在」？这个问题后面第六步会解决。

---

### 第五步：用 `len` 查数量

```go
fmt.Println("len:", len(m))
```

`len` 函数对 map 同样适用，返回当前 map 里有多少个键值对。现在 `m` 里有 `k1` 和 `k2` 两个键，所以 `len(m)` 返回 `2`。这个和数组、切片用法一样，很好记。

---

### 第六步：用 `delete` 删除键值对

```go
delete(m, "k2")
fmt.Println("map:", m)
```

`delete` 是内建函数，专门用来从 map 里删除一个键值对。第一个参数是 map，第二个参数是要删的键。删完之后 `m` 里就只剩 `k1:7` 了。

**注意**：如果你删一个根本不存在的键，`delete` 不会报错，静默处理，什么都不发生。这是 Go 的设计哲学之一：能不出错就不出错，让代码更健壮。

---

### 第七步：判断键是否存在——两值返回写法

```go
_, prs := m["k2"]
fmt.Println("prs:", prs)
```

这是 Go map 里最重要的惯用写法之一，一定要掌握！

map 取值其实可以返回两个值：第一个是键对应的值，第二个是一个 `bool`，表示「这个键在 map 里存不存在」。完整写法是：

```go
val, ok := m["k2"]
```

如果 `"k2"` 存在，`ok` 是 `true`，`val` 是对应的值；如果不存在，`ok` 是 `false`，`val` 是零值。

在这段代码里，因为我们刚才已经把 `"k2"` 删掉了，所以 `prs` 是 `false`。代码里用了**空白标识符（blank identifier）** `_` 来忽略第一个返回值（值本身），因为我们只关心键存不存在。`_` 是 Go 里的特殊占位符，意思是「我知道这里有个返回值，但我不需要，编译器也别给我报「变量未使用」的错」。

这个写法解决了第四步里说的歧义问题：不再纠结「0 是真的 0 还是不存在」，直接用 `ok` 来判断，清清楚楚。实际项目里，这个模式到处都是，长这样：

```go
if val, ok := m["someKey"]; ok {
    // 键存在，val 是真实的值
    fmt.Println(val)
} else {
    // 键不存在
    fmt.Println("没找到")
}
```

---

### 第八步：字面量初始化 map

```go
n := map[string]int{"foo": 1, "bar": 2}
fmt.Println("map:", n)
```

这是 map 的「字面量（literal）」初始化写法，可以在声明的同时就把数据填进去，一行搞定，不需要先 `make` 再赋值。格式是 `map[键类型]值类型{键1: 值1, 键2: 值2}`。

什么时候用 `make`，什么时候用字面量？——**如果初始数据已知，用字面量；如果要动态添加，用 `make`**。

---

### 总结一下 map 的全套操作

| 操作 | 写法 | 备注 |
|------|------|------|
| 创建空 map | `make(map[K]V)` | 必须 make，不然 nil map 写入会 panic |
| 写入/更新 | `m[key] = val` | 同键会覆盖 |
| 读取 | `val := m[key]` | 键不存在返回零值 |
| 判断键存在 | `val, ok := m[key]` | ok 是 bool |
| 删除 | `delete(m, key)` | 删不存在的键不会报错 |
| 数量 | `len(m)` | 返回键值对个数 |
| 字面量初始化 | `map[K]V{k1:v1}` | 声明时直接赋值 |

map 是日常写 Go 代码最频繁用到的结构之一，把这几个操作练熟，下一关我们用 `range` 来遍历它，更爽！

## 💡 师兄的碎碎念

- 别用 `var m map[string]int` 之后就直接赋值，这样 m 是 nil map，写入会直接 panic！一定要记得用 `make` 或字面量初始化。
- `val, ok := m[key]` 这个双返回值的写法，在面试里频繁出现，记住：`ok` 为 `false` 才说明 key 不存在，`val` 这时候是零值，不能靠它判断。
- map 是引用类型，把 map 传给函数，函数里的修改会影响原来的 map，这点和 slice 一样，要留意副作用。
- 遍历 map 的顺序在 Go 里是**不确定的**，每次跑输出顺序可能不一样，不要写依赖顺序的代码！

## 🎓 这一关的知识点清单

- **make 创建 map**：用 `make(map[key-type]val-type)` 创建空 map，这是最常见的初始化方式
- **字面量初始化**：可以用 `map[string]int{"foo": 1, "bar": 2}` 一行搞定声明和初始化
- **delete 删除键**：内建函数 `delete(m, key)` 直接移除键值对，不存在的 key 也不会 panic
- **双返回值检测 key 是否存在**：`val, ok := m[key]`，用 `ok` 区分「key 不存在」和「值恰好是零值」这两种情况，面试必考！
- **len 获取长度**：`len(m)` 返回当前键值对数量，和 slice 用法一致
