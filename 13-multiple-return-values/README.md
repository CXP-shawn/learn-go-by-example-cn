# 第 13 关：多返回值（师兄带你学 Go）

## 🎯 这一关你会学到

- Go 语言原生支持**多返回值**的语法
- 如何定义一个能返回多个值的函数
- 如何用**多赋值**操作来接收多个返回值
- 如何使用**空白标识符 `_`** 忽略不需要的返回值
- 多返回值在实际项目中最经典的使用场景：同时返回结果和错误信息

---

## 🤔 先想一个问题

假设你写了一个函数，需要同时告诉调用者两件事：计算结果是多少，以及有没有出错。在其他语言里你会怎么做？

- Java：抛异常，或者用一个对象封装两个字段
- Python：返回一个元组 `(result, error)`
- C：通过指针参数传出

Go 的解决方案更优雅——**原生支持多返回值**，语法简洁，无需任何包装。这也是 Go 语言设计哲学的体现：把常用的事情做得简单。

---

## 📖 看代码

```go
// Go 原生支持 _多返回值_。
// 这个特性在 Go 语言中经常用到，例如用来同时返回一个函数的结果和错误信息。

package main

import "fmt"

// `(int, int)` 在这个函数中标志着这个函数返回 2 个 `int`。
func vals() (int, int) {
	return 3, 7
}

func main() {

	// 这里我们通过 _多赋值_ 操作来使用这两个不同的返回值。
	a, b := vals()
	fmt.Println(a)
	fmt.Println(b)

	// 如果你仅仅需要返回值的一部分的话，你可以使用空白标识符 `_`。
	_, c := vals()
	fmt.Println(c)
}
```

## 🔍 师兄给你逐行拆

师兄带你把这段代码的每一个细节都说透。

### 函数返回多个值的语法

```go
func vals() (int, int) {
    return 3, 7
}
```

**函数签名解读：**

- `func vals()`：函数名是 `vals`，没有参数
- `(int, int)`：返回值类型列表，用括号包裹，多个返回值类型用逗号分隔

这里的关键变化是：当函数只有一个返回值时，返回值类型直接写，不需要括号（如 `func plus(a, b int) int`）。当有**多个返回值**时，必须用括号把所有返回值类型括起来。

```go
// 单返回值：不需要括号
func single() int { ... }

// 多返回值：必须用括号
func multiple() (int, string, bool) { ... }
```

**return 语句：**

```go
return 3, 7
```

多个返回值之间用逗号分隔。这里返回两个整数：3 和 7。顺序很重要！第一个值对应第一个返回类型，第二个值对应第二个返回类型。

### 接收多个返回值——多赋值操作

```go
a, b := vals()
fmt.Println(a)
fmt.Println(b)
```

调用 `vals()` 时，它返回两个值（3 和 7），我们用**多赋值语法** `a, b := ...` 同时接收它们。

- `a` 接收第一个返回值：3
- `b` 接收第二个返回值：7

这里用了 `:=` 短变量声明，所以 `a` 和 `b` 都是新变量，类型由编译器自动推导为 `int`。

`fmt.Println(a)` 打印 3，`fmt.Println(b)` 打印 7。

### 用空白标识符忽略部分返回值

```go
_, c := vals()
fmt.Println(c)
```

有时候函数返回多个值，但我们只需要其中一部分。这时用空白标识符 `_` 来占位，表示"这个返回值我不需要，帮我丢掉"。

这里我们只想要第二个返回值（7），所以第一个位置用 `_` 忽略。`c` 接收第二个返回值 7。

`fmt.Println(c)` 打印 7。

如果你需要第一个但不需要第二个，写法是 `a, _ := vals()`。

### 为什么不能直接忽略，而要用 `_`？

这是个好问题！Go 的设计要求：如果函数返回多个值，你在接收时必须为每个返回值提供一个变量或者 `_`。不能只写一个变量来接收多返回值的一部分，编译器会报错：

```go
// 错误！vals() 返回 2 个值，但这里只提供了 1 个变量
a := vals()  // 编译报错：multiple-value vals() used in single-value context
```

`_` 就是为了解决这个问题——让你可以有选择地接收需要的返回值。

### 多返回值在实际开发中的经典用法

Go 语言中最常见的多返回值模式是同时返回**结果和错误**：

```go
// 标准库中大量使用这种模式
file, err := os.Open("filename.txt")
if err != nil {
    // 处理错误
    fmt.Println("打开文件失败:", err)
    return
}
// 正常使用 file

n, err := fmt.Println("Hello")  // 返回写入字节数和错误
```

这是 Go 错误处理的核心模式。Go 不使用异常（exception），而是用多返回值让函数同时返回结果和可能的错误，调用者必须显式检查错误，这让错误处理变得清晰、可控。

### 命名返回值（扩展知识）

Go 还支持给返回值命名，这样在函数签名里就能看到每个返回值的语义：

```go
func minMax(arr []int) (min, max int) {
    min = arr[0]
    max = arr[0]
    for _, v := range arr {
        if v < min { min = v }
        if v > max { max = v }
    }
    return  // 裸 return，自动返回命名的返回值
}
```

命名返回值让函数的意图更清晰，但要注意裸 `return` 可能降低可读性，使用时要谨慎。

## 💡 师兄的碎碎念

**多返回值 vs 其他语言的解决方案**

在 Java 里，你通常需要创建一个包装类来返回多个值，或者抛异常来传递错误信息。在 Python 里虽然可以返回元组，但需要手动解包。Go 的多返回值从语言层面原生支持，不需要任何包装，是真正的语言特性。

**Go 为什么选择多返回值而不是异常？**

Go 的设计者认为，异常机制会导致控制流难以追踪，错误处理被隐藏在堆栈展开中，容易忘记处理。多返回值模式强迫调用者在每次函数调用时显式考虑错误，让错误处理成为代码的一部分，而不是可以忽略的"意外"。虽然写起来稍微啰嗦一点，但代码更可靠。
