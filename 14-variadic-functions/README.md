## 🤔 先想一个问题

你有没有好奇过，`fmt.Println` 为什么可以这样用？

```go
fmt.Println("hello")
fmt.Println("hello", "world")
fmt.Println(1, 2, 3, 4, 5)
```

传一个参数可以，传三个参数也可以，传五个参数还是可以。它到底是怎么做到「接受任意数量参数」的？

答案就是这一关的主角——**变参函数**。

Go 里用 `...类型` 这个魔法语法，就能让一个函数接受任意数量的同类型参数。听起来很神奇，但原理其实很简单，师兄这就带你彻底搞明白。

---

## 📖 看代码

```go
// [_可变参数函数_](https://zh.wikipedia.org/wiki/%E5%8F%AF%E8%AE%8A%E5%8F%83%E6%95%B0%E5%87%BD%E6%95%B8)。
// 在调用时可以传递任意数量的参数。
// 例如，`fmt.Println` 就是一个常见的变参函数。

package main

import "fmt"

// 这个函数接受任意数量的 `int` 作为参数。
func sum(nums ...int) {
	fmt.Print(nums, " ")
	total := 0
	for _, num := range nums {
		total += num
	}
	fmt.Println(total)
}

func main() {

	// 变参函数使用常规的调用方式，传入独立的参数。
	sum(1, 2)
	sum(1, 2, 3)

	// 如果你有一个含有多个值的 slice，想把它们作为参数使用，
	// 你需要这样调用 `func(slice...)`。
	nums := []int{1, 2, 3, 4}
	sum(nums...)
}
```

## 🔍 师兄给你逐行拆

好，代码不长，但每一行都有值得深挖的东西。师兄带你一行一行地过。

### 第一步：理解函数签名 `func sum(nums ...int)`

```go
func sum(nums ...int) {
```

这里的核心是 `...int` 这个语法。三个点 `...` 加上类型名，就表示「这个参数可以接受零个或多个该类型的值」。

你可能会问：`nums` 在函数体里到底是什么类型？答案是：**`[]int`，也就是一个 int 切片**。

这一点非常重要，也是很多新手一开始搞不清楚的地方。虽然调用者可以传入一个个独立的 `int` 值，但在函数内部，Go 语言会自动把这些值打包成一个切片交给你。所以你在函数里可以用所有切片的操作来处理它，比如 `len(nums)`、`nums[0]`、`range nums` 等等。

换句话说，`...int` 是给**调用者**看的语法糖，而函数内部拿到的永远是一个切片。这个设计既方便了调用者（不需要手动构造切片），又给了函数内部强大的处理能力（切片的所有操作都可以用）。

还有一点要注意：`...` 参数只能出现在参数列表的**最后一位**。你不能这样写：

```go
// 错误！...参数必须在最后
func badFunc(nums ...int, extra int) {}
```

但你可以这样写，在变参前面加普通参数：

```go
// 正确，prefix 是普通参数，nums 是变参
func printWithPrefix(prefix string, nums ...int) {}
```

---

### 第二步：函数体的第一行 `fmt.Print(nums, " ")`

```go
fmt.Print(nums, " ")
```

这里直接把 `nums` 打印出来了。因为 `nums` 本质上是一个 `[]int` 切片，所以打印出来的格式是 `[1 2]` 这样的形式，也就是方括号包裹、空格分隔的数字序列。

注意这里用的是 `fmt.Print` 而不是 `fmt.Println`，所以它不会自动换行。后面的 `" "` 是在数组和总和之间加一个空格，这样输出看起来更整齐。

---

### 第三步：累加逻辑

```go
total := 0
for _, num := range nums {
    total += num
}
```

这段代码非常经典。用 `range` 遍历切片（也就是变参 `nums`），把每个元素累加到 `total` 里。`_` 是用来忽略索引的，因为我们只关心值本身。

这里再次印证了前面说的：在函数内部，`nums` 就是一个普通的 `[]int` 切片，用 `range` 遍历它和遍历任何其他切片没有任何区别。

最后：

```go
fmt.Println(total)
```

打印出总和，并且换行。结合前面的 `fmt.Print(nums, " ")`，每次调用 `sum` 函数，输出的格式就是：`[元素们] 总和`，然后换行。

---

### 第四步：普通调用方式

```go
sum(1, 2)
sum(1, 2, 3)
```

这是最直接的调用方式：把参数一个个写出来，用逗号分隔。Go 会自动把 `1, 2` 打包成 `[]int{1, 2}` 传给函数。

- `sum(1, 2)` → `nums` 在函数内是 `[1 2]`，总和是 `3`
- `sum(1, 2, 3)` → `nums` 在函数内是 `[1 2 3]`，总和是 `6`

你甚至可以一个参数都不传：

```go
sum() // 合法！nums 是一个空切片 []int{}
```

这种情况下 `total` 还是 `0`，循环一次都不执行。这在某些场景下很有用，比如你的函数需要处理「可能没有参数」的情况。

---

### 第五步：切片展开语法 `sum(nums...)`

```go
nums := []int{1, 2, 3, 4}
sum(nums...)
```

**这是本关最重要、也是最容易被忽视的知识点。**

如果你已经有一个切片，想把它里面的所有元素传给变参函数，你不能直接写 `sum(nums)`，因为这样传的是一个切片对象，类型是 `[]int`，而 `sum` 的变参类型是 `int`，类型不匹配，编译器会报错。

你需要用 `nums...`，也就是在切片变量后面加三个点，这告诉 Go 编译器：「把这个切片里的元素一个一个展开，当作独立的参数传进去。」

这个操作有个专业名词叫做「**切片展开（slice spread）**」或者「**unpack**」，在 Python 里有类似的 `*args` 语法，在 JavaScript 里有 `...spread` 语法，Go 的 `...` 是同类概念的 Go 版本实现。

特别需要注意的是：`nums...` 这个语法**只能用于变参函数调用**，而且切片的元素类型必须和变参类型完全一致。下面这些是错误用法：

```go
// 错误：不能把 []string 展开给 ...int
words := []string{"a", "b"}
sum(words...) // 编译报错

// 错误：不能在非变参函数里用 ...
func add(a, b int) int { return a + b }
add(nums...) // 编译报错
```

---

### 性能小提示

当你用 `sum(1, 2, 3)` 这种方式调用时，Go 会在调用时分配一个临时切片 `[]int{1, 2, 3}` 来存放参数。而当你用 `sum(nums...)` 展开一个已有的切片时，Go **不会**创建新切片，而是直接把原切片传进去（底层共享同一块内存）。

这意味着：如果你在 `sum` 函数内部修改了 `nums` 的某个元素，调用方的 `nums` 也会被改变！这是一个很微妙的细节，在实际开发中要小心。

---

### 变参函数的实际应用场景

变参函数在 Go 标准库和日常开发中非常常见：

1. **日志函数**：`log.Printf(format, args...)` 接受格式化字符串和任意数量的参数。
2. **append 函数**：Go 内置的 `append(slice, elems...)` 本身就是一个变参函数。
3. **错误聚合**：有些框架会用 `func combineErrors(errs ...error) error` 来合并多个错误。
4. **配置选项模式（Functional Options）**：这是 Go 里非常流行的设计模式，用 `func New(opts ...Option)` 来传入可选配置。

## 💡 师兄的碎碎念

**碎碎念一：变参函数不是「任意类型」的魔法**

`...int` 只能接受 `int` 类型，不能同时接受 `int` 和 `string`。如果你想接受任意类型的参数，你需要用 `...interface{}`（Go 1.18 之前）或者 `...any`（Go 1.18 之后，`any` 是 `interface{}` 的别名）。`fmt.Println` 的签名其实就是 `func Println(a ...any) (n int, err error)`，这就是它能接受任何类型参数的原因。

**碎碎念二：`...` 的位置别搞错**

定义时，`...` 在类型前面：`func f(nums ...int)`。
调用时，`...` 在变量后面：`f(nums...)`。
两个地方的 `...` 意思截然不同，一定要区分清楚。

**碎碎念三：变参函数是切片的语法糖，别神秘化它**

理解了「变参本质是切片」这一点，你就不会对变参函数感到陌生了。它只是 Go 为了让函数调用更方便而提供的一层语法包装，底层没有任何黑魔法。

**碎碎念四：append 是内置变参函数的最好例子**

```go
s := []int{1, 2, 3}
s = append(s, 4)        // 追加单个元素
s = append(s, 5, 6, 7) // 追加多个元素

other := []int{8, 9, 10}
s = append(s, other...) // 追加一个切片的所有元素
```

`append` 的第二个参数就是 `...int`（对于 `[]int` 切片），所以你既可以一个一个地传，也可以用 `...` 把另一个切片展开传进去。这是 Go 里最高频使用变参展开语法的场景，一定要熟练掌握。

## 🎓 这一关的知识点清单

| 知识点 | 说明 |
|--------|------|
| `func f(params ...T)` | 定义变参函数，`params` 在函数内是 `[]T` 类型 |
| 变参本质 | 变参在函数内部是一个切片，支持所有切片操作 |
| 普通调用 | `f(1, 2, 3)` 逐个传入参数，Go 自动打包成切片 |
| 切片展开调用 | `f(slice...)` 把已有切片展开为独立参数传入 |
| 变参位置限制 | `...` 参数只能出现在参数列表最后一位 |
| 零参数调用 | `f()` 合法，`params` 是空切片 `[]T{}` |
| `...any` | 接受任意类型的变参，`fmt.Println` 就是这样实现的 |
| 性能细节 | `f(slice...)` 不会复制切片，函数内修改会影响原切片 |
| 常见应用 | `append`、`fmt.Printf`、日志函数、Functional Options 模式 |
