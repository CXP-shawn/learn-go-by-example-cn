# 第 8 关：数组（师兄带你学 Go）

## 🎯 这一关你会学到

你会搞清楚 Go 里的**数组**是什么、怎么声明、怎么读写，还有怎么用嵌套循环处理二维数组。数组是 Go 里最基础的数据容器，学好了它，后面的切片（Slice）就会顺滑很多。

## 🤔 先想一个问题

你去奶茶店买了一排 5 杯奶茶，店员把它们整整齐齐放进一个只能装 5 杯的托盘里，每个格子都有编号：0 号、1 号、2 号……每个格子只能放一杯，多了塞不进去，少了就空着。

这个托盘，就是 Go 里的**数组（Array）**——一种**长度固定、类型统一、按顺序编号存放元素的数据结构**。数组一旦创建，格子数量就定死了，不能动态扩容。想装更多？那是下一关切片（Slice）干的事。

## 📖 看代码

```go
// 在 Go 中，_数组_ 是一个具有编号且长度固定的元素序列。

package main

import "fmt"

func main() {

 // 这里我们创建了一个刚好可以存放 5 个 `int` 元素的数组 `a`。
 // 元素的类型和长度都是数组类型的一部分。
 // 数组默认值是零值，对于 `int` 数组来说，元素的零值是 `0`。
 var a [5]int
 fmt.Println("emp:", a)

 // 我们可以使用 `array[index] = value` 语法来设置数组指定位置的值，
 // 或者用 `array[index]` 得到值。
 a[4] = 100
 fmt.Println("set:", a)
 fmt.Println("get:", a[4])

 // 内置函数 `len` 可以返回数组的长度。
 fmt.Println("len:", len(a))

 // 使用这个语法在一行内声明并初始化一个数组。
 b := [5]int{1, 2, 3, 4, 5}
 fmt.Println("dcl:", b)

 // 数组类型是一维的，但是你可以组合构造多维的数据结构。
 var twoD [2][3]int
 for i := 0; i < 2; i++ {
 for j := 0; j < 3; j++ {
 twoD[i][j] = i + j
 }
 }
 fmt.Println("2d: ", twoD)
}
```

## 🔍 师兄给你逐行拆

### 第一步：声明一个什么都没放的数组

```go
var a [5]int
fmt.Println("emp:", a)
```

这两行是在说：「给我一个能装 5 个整数的格子托盘，先不放东西。」

`var a [5]int` 这里有三个要素拆开来看：
- `var` 是声明关键字，告诉编译器「我要定义一个变量」。
- `a` 是变量名，随便取，叫 `tray`、`box` 都行。
- `[5]int` 是**数组类型**，意思是「能装 5 个 int 的数组」，注意 **长度 5 是类型的一部分**，`[5]int` 和 `[6]int` 是两种完全不同的类型，就像 5 格托盘和 6 格托盘根本不通用一样。

这里要特别讲一个概念：**零值（Zero Value）**，就是「变量声明了但没有手动赋值时，Go 自动帮你填的默认值」。对于 `int` 类型，零值是 `0`。所以 `var a [5]int` 之后，数组里 5 个格子全是 `0`，打印出来就是 `emp: [0 0 0 0 0]`。

这个设计很贴心——Go 不允许你用到「垃圾值」，所有变量声明后都有确定的初始状态，不像某些语言声明了不初始化就是随机内存垃圾，新手经常被坑。

---

### 第二步：往格子里放东西，再取出来

```go
a[4] = 100
fmt.Println("set:", a)
fmt.Println("get:", a[4])
```

`a[4] = 100` 意思是「把托盘第 4 号格子里放上 100 这杯奶茶」。

这里有个超级经典的坑，师兄重点提醒：**Go 数组的下标（Index）从 0 开始**，所以一个长度为 5 的数组，合法下标是 `0、1、2、3、4`，没有 `5`！`a[4]` 访问的是**第 5 个**元素，而不是第 4 个。

新手最容易犯的错叫做**越界访问（Index Out of Range）**：比如你写了 `a[5] = 100`，Go 编译器在编译期如果能检测到会直接报错；如果是运行时动态下标越界，程序会直接 panic 崩掉，不会假装没事。所以数组下标要时刻留意「最大合法值 = 长度 - 1」。

`a[4] = 100` 之后，数组从 `[0 0 0 0 0]` 变成了 `[0 0 0 0 100]`，只有最后那个格子被改动了。

`fmt.Println("get:", a[4])` 则是把第 4 号格子的值读出来打印，输出 `get: 100`。读取语法和赋值语法对称，很好记：赋值是 `a[i] = 值`，取值是 `a[i]`。

---

### 第三步：用 len 查数组长度

```go
fmt.Println("len:", len(a))
```

`len` 是 Go 的**内置函数（Built-in Function）**，直接用，不需要 import 任何包。它返回数组里格子的总数，这里打印出来是 `len: 5`。

你可能会问：「我自己写的 `[5]int`，长度不是已经知道了吗，为什么还要 `len`？」实际开发中数组往往通过函数参数传递，或者被封装起来，这时候 `len` 就很有用。更重要的是，养成用 `len` 的习惯，在后面的切片和循环控制里会非常频繁。

---

### 第四步：声明的同时初始化

```go
b := [5]int{1, 2, 3, 4, 5}
fmt.Println("dcl:", b)
```

这是更常见的写法，一行搞定「声明 + 赋初始值」。`:=` 是 Go 的**短变量声明（Short Variable Declaration）**，等号右边 `[5]int{1, 2, 3, 4, 5}` 叫做**数组字面量（Array Literal）**，花括号里按顺序列出每个格子的初始值。

类比一下：就像你在奶茶店直接说「给我一杯珍珠、一杯椰果、一杯布丁、一杯红豆、一杯芋圆」，店员直接按顺序装好，不需要你一杯一杯自己放。

打印结果是 `dcl: [1 2 3 4 5]`，非常直观。

注意：花括号里的元素个数必须和 `[5]` 一致，少了或多了编译器都会报错。如果你不想数，可以用 `[...]int{1, 2, 3, 4, 5}`，三个点让编译器自己去数，但这关例子里用的是明确的 `[5]`，知道有这个语法就好。

---

### 第五步：二维数组——托盘里套托盘

```go
var twoD [2][3]int
for i := 0; i < 2; i++ {
    for j := 0; j < 3; j++ {
        twoD[i][j] = i + j
    }
}
fmt.Println("2d: ", twoD)
```

`[2][3]int` 是**二维数组（Two-Dimensional Array）**，可以理解为「2 行 3 列的整数表格」，或者说「2 个各自能装 3 个 int 的小托盘，拼在一起形成的大托盘」。

外层循环 `for i := 0; i < 2; i++` 遍历行（0 和 1），内层循环 `for j := 0; j < 3; j++` 遍历列（0、1、2），`twoD[i][j] = i + j` 把行号加列号的结果存进去。

实际填入的值：
- `twoD[0][0] = 0+0 = 0`
- `twoD[0][1] = 0+1 = 1`
- `twoD[0][2] = 0+2 = 2`
- `twoD[1][0] = 1+0 = 1`
- `twoD[1][1] = 1+1 = 2`
- `twoD[1][2] = 1+2 = 3`

打印结果是 `2d:  [[0 1 2] [1 2 3]]`，外层是行，内层是列，非常清晰。

二维数组在实际开发里可以用来表示棋盘、矩阵、像素网格等等。虽然日常业务代码里用得不算多，但理解它的结构对后续理解切片的切片会很有帮助。

---

### 总结几个容易踩的坑

1. **下标从 0 开始，最大下标 = 长度 - 1**，访问 `a[5]` 对于 `[5]int` 就是越界，轻则编译报错，重则运行时 panic。
2. **数组长度是类型的一部分**，`[5]int` 和 `[6]int` 是不同类型，不能互相赋值，传参时也要严格匹配。
3. **Go 数组是值类型**，把一个数组赋给另一个变量，是完整复制一份，修改新的不会影响旧的。这和很多其他语言的引用语义不同，是新手经常感到意外的地方。
4. 数组长度固定、不能动态扩容——如果你的需求是「不确定要装多少」，那应该用下一关的**切片（Slice）**。

## 💡 师兄的碎碎念

- 🔒 **数组是值类型**，这点很容易踩坑！把数组传给函数或赋值给另一个变量时，Go 会完整地复制一份，修改副本不会影响原数组。如果数组很大，建议传指针或改用切片，省内存也省性能。
- 🤔 **`[...]int{1,2,3}` 这个省略号语法**超级好用，不用自己数元素个数，让编译器帮你算，初始化长数组时强烈推荐。
- ⚠️ **数组长度是类型的一部分**，所以函数参数如果写 `func foo(arr [5]int)`，传一个 `[3]int` 进去编译直接报错。师兄当年刚学 Go 时被这个坑了好几次，切记！
- 🧩 **二维数组的遍历顺序**：外层循环控制行（第一维），内层控制列（第二维），和数学里矩阵的行列顺序一致，不用特别记，按直觉来就对了。

## 🎓 这一关的知识点清单

- **数组声明与类型**：Go 数组的类型包含元素类型和长度两部分，`[5]int` 和 `[3]int` 是完全不同的类型，不可互相赋值，面试常考这个细节。
- **零值初始化**：声明数组时无需手动清零，Go 会自动将所有元素初始化为对应类型的零值（int → 0，bool → false，string → ""）。
- **字面量初始化**：使用 `b := [5]int{1, 2, 3, 4, 5}` 可以在一行内声明并赋初值；也可以用 `[...]int{1, 2, 3}` 让编译器自动推断长度。
- **len 函数**：内置 `len()` 返回数组长度，对于数组来说长度是编译期常量，可以用在需要常量的地方。
- **多维数组**：Go 通过嵌套一维数组实现多维结构，`[2][3]int` 表示 2 行 3 列的二维 int 数组，按行优先存储。
