# 第 20 关：方法（师兄带你学 Go）

## 🎯 这一关你会学到

这一关你会学会怎么给 struct 定义方法，核心是搞清楚值接收者和指针接收者的区别：值接收者拿到的是副本，修改不影响原始数据；指针接收者拿到的是地址，可以直接改原对象。Go 还会自动帮你解引用，所以不管你手里拿的是值还是指针，调用方法的写法都一样，编译器帮你处理细节。

## 🤔 先想一个问题

打开外卖 app，点进一个订单，你会看到一个。 App 把数据（订单）和它能做的事（取消、评价、再来一单）绑在一起。这就是编程里"方法"的直觉：数据不孤立存在，它自带一些能操作它自己的动作。

**方法是绑定到某个类型上的函数，通过接收者语法 `func (r Type) Name() ...` 定义，用 `实例.方法()` 调用；接收者可以是值或指针，分别对应复制接收者 vs 修改接收者。**

## 📖 看代码

```go
// Go 支持为结构体类型定义_方法(methods)_ 。

package main

import "fmt"

type rect struct {
	width, height int
}

// 这里的 `area` 是一个拥有 `*rect` 类型接收器(receiver)的方法。
func (r *rect) area() int {
	return r.width * r.height
}

// 可以为值类型或者指针类型的接收者定义方法。
// 这是一个值类型接收者的例子。
func (r rect) perim() int {
	return 2*r.width + 2*r.height
}

func main() {
	r := rect{width: 10, height: 5}

	// 这里我们调用上面为结构体定义的两个方法。
	fmt.Println("area: ", r.area())
	fmt.Println("perim:", r.perim())

	// 调用方法时，Go 会自动处理值和指针之间的转换。
	// 想要避免在调用方法时产生一个拷贝，或者想让方法可以修改接受结构体的值，
	// 你都可以使用指针来调用方法。
	rp := &r
	fmt.Println("area: ", rp.area())
	fmt.Println("perim:", rp.perim())
}
```

## 🔍 师兄给你逐行拆

### 指针接收者 vs 值接收者：两种写法的差异

来看两个方法的定义：

```go
func (r *rect) area() int  { return r.width * r.height }
func (r rect)  perim() int { return 2*r.width + 2*r.height }
```

`area` 使用 **指针接收者** `*rect`，`r` 是原变量的地址，在方法内修改 `r.width` 等字段，调用方的变量会同步变化。`perim` 使用 **值接收者** `rect`，Go 会把结构体复制一份传进来，方法内的任何修改都只影响这个副本，原变量纹丝不动。

**如何选择？** 遵循以下经验法则：

- 需要修改接收者的字段 → 用指针接收者；
- 接收者是较大的结构体，想避免每次调用都拷贝 → 用指针接收者；
- 方法只读、结构体本身很小 → 用值接收者即可。

还有一条一致性原则：**同一类型的所有方法，尽量统一使用指针或值，不要混用**。混用会让接口实现变得微妙——值类型只能满足值接收者的方法集，指针类型才能同时满足两者，稍不注意就会出现编译错误。保持统一，代码更易维护。

### 调用方法：. 语法通吃值和指针

```go
r.area()   // r 是值，area 接收者是 *rect，Go 自动取地址 → (&r).area()
rp.area()  // rp 已是指针，直接传入
r.perim()  // perim 接收者是 rect，Go 自动解引用 → (*rp).perim()
rp.perim() // rp 是指针，Go 自动解引用
```

Go 的 `.` 语法非常智能：**调用方法时，Go 会自动处理 `&` 和 `*`**，无需手动转换。

- 接收者是 `*rect`，但你持有的是值 `r`？Go 自动执行 `&r`。
- 接收者是 `rect`，但你持有的是指针 `rp`？Go 自动执行 `*rp`。

**类比**：外卖小哥既能接过实体餐盒（值），也能接过储物柜密码（指针）——他都会根据实际情况正确配送，你无需关心内部细节。

这一特性让代码更简洁，调用方不必纠结「我现在拿的是值还是指针」，专注业务逻辑即可。

### 方法的本质是带接收者的函数

在 Go 中，方法和函数并没有本质区别。

Go 这么设计的好处：调用读起来更自然 `r.area()` 而不是 `area(r)`；并为接口系统打下基础（接口通过方法集判断）。

**坑**：Go 方法**不能定义在其他包的类型上**（包括标准库 int/string），要"扩展"只能自己包装一层：`type MyInt int; func (m MyInt) Foo(){}`。

### 方法集：为接口打基础

一个类型所有方法的集合叫**方法集**。规则很直接：值类型 `T` 的方法集只包含**值接收者**的方法；指针类型 `*T` 的方法集则同时包含值接收者和指针接收者的方法。

```go
type Counter struct{ n int }
func (c Counter) Value() int  { return c.n }   // 值接收者
func (c *Counter) Inc()       { c.n++ }        // 指针接收者
// *Counter 的方法集：Value + Inc
// Counter  的方法集：仅 Value
```

这个区别在下一关 **Interfaces** 中至关重要：**一个类型是否实现了某个接口，看的就是它的方法集是否包含接口要求的所有方法**。如果接口要求 `Inc()`，那么 `Counter` 值无法满足，`*Counter` 才可以。方法不是孤立的语法糖——它是 Go 类型系统的根基。

## 🏃 跑一下试试

```bash
$ go run methods.go
area:  50
perim: 30
area:  50
perim: 30
```

两次输出完全一致——Go 在调用时会自动处理值与指针之间的转换，无论通过值变量还是指针变量调用方法，结果相同。

## 💡 师兄的碎碎念

- **保持接收者风格一致**：同一类型的所有方法应统一使用值接收者或指针接收者，混用会让方法集规则变得复杂难以预测。
- **nil 指针调用需谨慎**：指针接收者方法若在 `nil` 指针上调用且方法内部访问了字段，会触发 panic，使用前应做 nil 检查。
- **大 struct 优先用指针接收者**：值接收者每次调用都会拷贝整个结构体，字段较多时对性能有明显影响，应改用 `*T`。
- **接收者形参名用类型首字母**：惯用写法如 `func (r *rect) area()`，避免使用 `self` 或 `this`，保持 Go 风格简洁一致。

## 🎓 这一关的知识点清单

- **方法定义语法**：`func (接收者) 方法名(参数) 返回值 {}` 是否已掌握，与普通函数的区别是否清晰？
- **指针接收者**：使用 `*T` 作为接收者时，方法内修改字段是否能反映到原始值？
- **值接收者**：使用 `T` 作为接收者时，方法操作的是副本，是否理解其不可变语义？
- **方法自动解引用**：通过变量或指针调用方法时，Go 是否自动完成取地址 / 解引用，行为是否符合预期？
- **方法集**：值类型与指针类型各自拥有的方法集范围是否理解，是否清楚为何指针类型能调用值接收者方法但反之不总成立？
