# 第 3 关：Variables（师兄带你学 Go）

## 🎯 这一关你会学到

这一关你会搞清楚 Go 里「变量」到底怎么声明、怎么用，以及 Go 编译器为啥这么

## 🤔 先想一个问题

想象你去奶茶店点单，收银员在点单系统里给你开一个「订单格子」，这个格子必须先告诉系统「这格放啥类型的东西」——是饮料名、杯数还是备注。你不能先开格子、再想放啥。Go 里的变量就是这个逻辑：**变量必须先声明（告诉编译器格子存在、放啥类型），才能用**。这不是 Go 在刁难你，而是编译器要在你程序跑起来之前就帮你检查「你往字符串格子里塞了个数字」这种低级错误——提前报错，总比上线崩溃要强得多。

## 📖 看代码

```go
// 在 Go 中，_变量_ 需要显式声明，并且在函数调用等情况下，
// 编译器会检查其类型的正确性。

package main

import "fmt"

func main() {

 // `var` 声明 1 个或者多个变量。
 var a = "initial"
 fmt.Println(a)

 // 你可以一次性声明多个变量。
 var b, c int = 1, 2
 fmt.Println(b, c)

 // Go 会自动推断已经有初始值的变量的类型。
 var d = true
 fmt.Println(d)

 // 声明后却没有给出对应的初始值时，变量将会初始化为 _零值_ 。
 // 例如，`int` 的零值是 `0`。
 var e int
 fmt.Println(e)

 // `:=` 语法是声明并初始化变量的简写，
 // 例如 `var f string = "short"` 可以简写为右边这样。
 f := "short"
 fmt.Println(f)
}
```

## 🔍 师兄给你逐行拆

### `var a = "initial"` —— 最基础的声明方式

`var` 是关键字，相当于跟编译器说「我要开一个格子」。`a` 是格子的名字，`"initial"` 是初始值。注意这里**没有写类型**，Go 编译器自己看右边的值推断出「哦这是个 string」。这叫**类型推断**。很多同学第一次看到会问：「那 Go 不是强类型语言吗？」——是的，它是，只是编译器帮你省了手打类型的力气，背后类型该是啥还是啥，你无法把 `a` 后面再赋值成一个 `int`，试试看编译器会当场翻脸。

### `var b, c int = 1, 2` —— 一口气声明多个变量

这行是「批量开格子」。两个格子 `b` 和 `c`，类型都是 `int`，初始值分别是 `1` 和 `2`。类比成宿舍申请两个储物柜，贴好标签、放好东西，一次性搞定。**注意**：这里类型 `int` 写在变量名后面，这是 Go 的语法习惯——「名字在前，类型在后」，和 C/Java 的「类型在前」刚好反过来。刚转过来的同学经常手滑写成 `int b, c`，记住调换一下。

### `var d = true` —— 布尔类型推断

同样是类型推断，右边给的是 `true`，编译器推断出 `d` 是 `bool` 类型。没啥特别的，就是提醒你：推断不只对字符串有效，数字、布尔、浮点数都能推断，Go 编译器挺聪明的。

### `var e int` —— 零值：Go 的「默认出厂设置」

这行声明了 `e`，但**没给初始值**。这在很多语言里是危险操作（C 里未初始化的变量是随机垃圾值，能坑哭你）。Go 直接规定：没给初始值，就用**零值**填充——`int` 的零值是 `0`，`string` 是 `""`，`bool` 是 `false`，指针是 `nil`。你可以把零值理解成「格子出厂时自带的默认填充物」，保证格子永远不是空的。这个设计让 Go 代码更安全，少了一类「读到垃圾数据」的 bug。

### `f := "short"` —— 短变量声明，Go 最常见的写法

`:=` 是 Go 里最高频的变量声明方式，**声明 + 赋值二合一**，而且只能在函数体内用（包级别变量不能用它）。它等价于 `var f string = "short"`，但少打好多字。类比成奶茶店的「常客快捷下单」——老顾客直接说「老样子」，不用一项一项报。

**最容易踩的坑**：`:=` 左边至少要有一个**新变量**，如果你想给已经声明过的变量重新赋值，直接用 `=` 就好，别再加冒号，否则编译器报「no new variables on left side of :=」。另外，`:=` 声明的变量必须被用到，Go 编译器对「声明了但没用的变量」是零容忍直接报错——这也是 Go 逼你写干净代码的小手段之一。

## 💡 师兄的碎碎念

- `:=` 是 Go 里最常见的变量声明方式，简洁到位，但只能用在**函数内部**，包级别的变量老老实实用 `var`，别想偷懒。
- Go 的**零值机制**是个好东西，`int` 是 `0`，`string` 是 `""`，`bool` 是 `false`，`pointer` 是 `nil`——不像某些语言给你来个

## 🎓 这一关的知识点清单

- **var 声明**：`var` 是最正式的声明方式，可以一次声明一个或多个变量，也可以指定类型，也可以让编译器推断，就像填表格可以填详细版也可以填简化版。
- **类型推断**：Go 编译器会根据初始值自动推断变量类型，`var d = true` 编译器自己知道这是 `bool`，不用你多嘴。
