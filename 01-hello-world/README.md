# 第 1 关：Hello World（师兄带你学 Go）

## 🎯 这一关你会学到

这一关你会写出人生第一个 Go 程序，搞清楚一个 Go 文件的「最小骨架」长什么样。你会理解 `package`、`import`、`func main()` 这三件套各自扮演什么角色，以及为什么少一样程序就跑不起来。

## 🤔 先想一个问题

想象你去奶茶店打工第一天，店长给你一张操作手册，上面写着：**「所有员工必须先打卡签到，再去领取工具，最后才能开始接单」**。少了任何一步，系统就报错，你就得被请出去。

Go 程序也是一样的脾气——它要求你必须按固定格式「报到」：先声明自己属于哪个包（`package`），再说明要用哪些外部工具（`import`），最后在规定的入口函数（`func main()`）里写真正的逻辑。这套顺序不能乱，少一步编译器就罢工。

所谓「Hello World」，就是程序界约定俗成的「第一声问候」——验证你的环境没问题、代码能跑起来，仅此而已。别小看这几行，它背后藏着 Go 程序运行的完整最小单元。

## 📖 看代码

```go
// 我们的第一个程序将打印传说中的"hello world"，
// 右边是完整的程序代码。

package main

import "fmt"

func main() {
 fmt.Println("hello world")
}
```

## 🔍 师兄给你逐行拆

### `package main` —— 你在哪个部门上班？

每个 Go 源文件的第一行必须声明自己属于哪个「包（package）」。包就像公司里的部门，负责把相关代码归在一起。

`main` 这个名字很特殊——Go 规定，**可以被直接运行的程序，必须有且只有一个 `package main`**。你可以把它理解成「前台接待部门」，外面的世界只认识它，其他包都是内部部门，外人进不来。

如果你把这行改成 `package foo`，代码照样能编译，但你就再也没法用 `go run` 直接跑它了——因为 Go 找不到「前台」。这是新手最容易懵的第一个坑。

---

### `import "fmt"` —— 去仓库领工具

`import` 就是去「工具仓库」领东西。`fmt` 是 Go 标准库里的一个包，全名是 **format（格式化）** 的缩写，专门负责和终端打交道——打印文字、读取输入、格式化字符串，都归它管。

你可以把它想象成奶茶店里的「封杯机」——不是每个动作都要自己造一台，直接从仓库推过来用就行。

有一个让新手很崩溃的规定：**import 进来的包，必须用到，否则编译直接报错**。Go 编译器在这点上极度「洁癖」，绝对不允许「领了工具放那儿落灰」。所以别手贱多 import 一个包然后不用，后果就是红色报错迎面扑来。

---

### `func main()` —— 程序的「开门营业」按钮

`func` 是 function（函数）的缩写，`main` 是这个函数的名字。`func main()` 就是 Go 程序的**唯一合法入口**——程序启动后，第一件事就是找到这个函数，然后从大括号 `{` 开始一行一行往下执行。

类比一下：你开奶茶店，不管后厨多热闹、备料多充分，顾客进门第一个接触的永远是前台。`main` 函数就是那个前台，整个程序的故事从这里开始。

注意大括号 `{` 必须和 `func main()` 在**同一行**，不能换行写成下一行——这是 Go 语法硬性要求，跟某些语言的习惯不一样，刚上手很容易犯。

---

### `fmt.Println("hello world")` —— 终于做了点实事

这才是整个程序唯一「干活」的一行。`fmt.Println` 的意思是：用 `fmt` 这个包里的 `Println` 函数，把括号里的内容**打印到终端，并自动换行**。

`Print` + `ln` = Print Line，打印一行。如果你用 `fmt.Print`（没有 ln），内容打印完不会换行，下一次输出会紧接着贴上去——看起来挤在一起，通常不是你想要的效果。

双引号里的 `"hello world"` 是一个**字符串字面量**，Go 里字符串必须用双引号包裹，单引号是给单个字符（rune）用的，两者不能混用，否则又是一个编译报错。

---

### 整体串一遍

四行代码，构成了 Go 程序能跑起来的**最小合法骨架**：声明包 → 引入工具 → 定义入口 → 执行逻辑。缺任何一环，Go 编译器都会礼貌但坚定地拒绝你。把这个结构刻进肌肉记忆里，后面所有的 Go 程序都是在这个骨架上「长肉」。

## 🏃 跑一下试试

```bash
# 第一步：直接用 go run 跑源码，省去手动编译
$ go run hello-world.go
hello world

# 第二步：用 go build 编译成本地可执行文件
$ go build hello-world.go

# 第三步：看看编译产物（Linux/macOS 叫 hello-world，Windows 是 hello-world.exe）
$ ls
hello-world    hello-world.go

# 第四步：直接运行二进制文件，不再需要 Go 环境
$ ./hello-world
hello world
```

## 💡 师兄的碎碎念

- `go run` 背后其实也做了编译，只是把中间产物扔到临时目录里，跑完自动清理——就像外卖小哥送完餐顺手带走了垃圾袋，你感觉不到而已。
- Go 文件必须以 `.go` 结尾，文件名用小写加连字符（`hello-world.go`）是社区惯例，别学 Java 那一套驼峰文件名，同事会用奇怪的眼神看你。
- `fmt.Println` 末尾自动加换行，如果你不想换行可以用 `fmt.Print`，想格式化输出就用 `fmt.Printf`——三兄弟各有分工，别记混了。
- 编译后的二进制文件**不依赖 Go 运行时环境**，直接扔给运维就能跑，这是 Go 相比 Python/Java 最让运维小哥感动的地方之一。

## 🎓 这一关的知识点清单

- **package main**：Go 程序的入口包，必须叫 `main`，就像你宿舍门牌必须写对才能收到快递。
- **import**：导入标准库或第三方包，`"fmt"` 是格式化 I/O 包，负责一切打印输出的体力活。
- **func main()**：程序执行的起点，整个 Go 程序从这里开始跑，一个可执行程序有且仅有一个 `main` 函数。
- **go run vs go build**：`go run` 是「边编译边跑」的快捷方式，适合开发调试；`go build` 生成真正的可执行二进制文件，部署上线用这个。
- **fmt.Println**：向标准输出打印一行文字并自动换行，`Println` 里的大写 `P` 不能写成小写，Go 用首字母大小写区分公开/私有。
