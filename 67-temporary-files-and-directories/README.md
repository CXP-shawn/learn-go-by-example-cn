# 第 67 关：临时文件和目录（师兄带你学 Go）

## 🎯 这一关你会学到

掌握 os.CreateTemp 和 os.MkdirTemp 两个生成临时文件/目录的神器：自动分配唯一名字、自动放进系统临时目录、搭配 defer 实现"用完即焚"，单元测试和中间数据处理的利器。

## 🤔 先想一个问题

你写过单元测试吗？每次都要准备一个测试文件、跑完还要记得手动删除，一不留神就在项目里留一堆垃圾文件被同事吐槽。Go 标准库给你准备了 os.CreateTemp/MkdirTemp，生成唯一名字的临时文件和目录，再配上 defer os.Remove/RemoveAll，就是一气呵成的优雅姿势。

## 📖 看代码

```go
// 在程序运行时，我们经常创建一些运行时用到，程序结束后就不再使用的数据。
// *临时目录和文件* 对于上面的情况很有用，因为它不会随着时间的推移而污染文件系统。

package main

import (
	"fmt"
	"os"
	"path/filepath"
)

func check(e error) {
	if e != nil {
		panic(e)
	}
}

func main() {

	// 创建临时文件最简单的方法是调用 `os.CreateTemp` 函数。
	// 它会创建并打开文件，我们可以对文件进行读写。
	// 函数的第一个参数传 `""`，`os.CreateTemp` 会在操作系统的默认位置下创建该文件。
	f, err := os.CreateTemp("", "sample")
	check(err)

	// 打印临时文件的名称。
	// 文件名以 `os.CreateTemp` 函数的第二个参数作为前缀，
	// 剩余的部分会自动生成，以确保并发调用时，生成不重复的文件名。
	// 在类 Unix 操作系统下，临时目录一般是 `/tmp`。
	fmt.Println("Temp file name:", f.Name())

	// defer 删除该文件。
	// 尽管操作系统会自动在某个时间清理临时文件，但手动清理是一个好习惯。
	defer os.Remove(f.Name())

	// 我们可以向文件写入一些数据。
	_, err = f.Write([]byte{1, 2, 3, 4})
	check(err)

	// 如果需要写入多个临时文件，最好是为其创建一个临时 *目录* 。
	// `os.MkdirTemp` 的参数与 `CreateTemp` 相同，
	// 但是它返回的是一个 *目录名* ，而不是一个打开的文件。
	dname, err := os.MkdirTemp("", "sampledir")
	fmt.Println("Temp dir name:", dname)

	defer os.RemoveAll(dname)

	// 现在，我们可以通过拼接临时目录和临时文件合成完整的临时文件路径，并写入数据。
	fname := filepath.Join(dname, "file1")
	err = os.WriteFile(fname, []byte{1, 2}, 0666)
	check(err)
}
```

## 🔍 师兄给你逐行拆

好，今天来聊聊 os.CreateTemp 这个函数。很多同学写业务代码的时候压根不会想到临时文件，但它其实是个很实用的工具，用好了能省不少事。

先说说临时文件到底有什么用。你在做数据处理的时候，有时候中间结果太大，不想全部塞进内存，这时候可以先写到临时文件里，处理完再读回来。写单元测试的时候，你需要一个真实存在的文件作为测试夹具，总不能每次都在项目目录里手动创建一个吧，用完还得记得删掉，麻烦死了。还有下载缓存这种场景，先把数据流写到临时文件，校验完整性之后再移动到正式位置，这样即使下载到一半程序崩了，也不会留下一个残缺的正式文件污染你的文件系统。这些场景都特别适合用临时文件来处理，用完就扔，干净利落。

好，来看这个函数的签名。os.CreateTemp 接收两个参数，第一个是父目录，第二个是文件名的模式字符串，返回一个已经打开的文件指针和一个错误值。

第一个参数，父目录，如果你传空字符串，Go 会帮你用系统默认的临时目录。不同操作系统的默认临时目录位置不一样。Linux 上就是根目录下的 tmp 目录，也就是斜杠 tmp 那个。Mac 稍微复杂一点，它在 var 下面有一层 folders 层级的目录，路径看起来有点奇怪，但你不用管，传空字符串 Go 自动处理。Windows 呢，是在用户的 AppData 下面的 Temp 目录里。总之，传空字符串是最常见的做法，让操作系统决定放在哪里，你不用操心路径兼容性问题，跨平台代码写起来也省心。当然如果你有特殊需求，比如你明确要把临时文件放在某个特定位置，也可以自己传一个目录路径进去。

第二个参数是 pattern，可以理解成文件名的前缀模板。比如你传一个字符串叫做 myapp-cache，Go 在创建文件的时候会在这个前缀后面自动拼上一串随机的字符作为后缀，生成类似 myapp-cache123456789 这样的文件名。这个机制有个很重要的意义，就是保证并发安全。你的程序如果同时有多个 goroutine 都在调用 os.CreateTemp，每次生成的文件名都是唯一的，不会互相覆盖，不会产生竞态条件。这一点比你自己拼时间戳或者随机数要靠谱得多，因为 Go 内部帮你处理了这些细节。

返回值这里要注意，拿到的文件指针是已经打开的状态，不需要你再调用 os.Open 去打开一次，可以直接写数据进去。用完之后记得调用 Close 方法关闭文件句柄，这是基本操作，不多说。

然后说一个特别有用的方法，就是在拿到文件指针之后调用它的 Name 方法，可以拿到这个临时文件的完整绝对路径。这个路径有时候很有用，比如你要把这个文件路径传给另一个函数处理，或者你要在日志里记录一下这个临时文件的位置方便调试，都可以用 Name 方法取出来。

关于清理，这里要强调一下，用完临时文件之后最好手动调用 os.Remove 把它删掉。很多同学会想，临时目录操作系统不是会自己清理吗，何必多此一举。这个想法是有问题的。操作系统确实会定期清理临时目录，但

好，咱们继续聊临时文件和目录的第二段——这次主角换成临时**目录**。

---

**os.MkdirTemp：一次给你一个专属临时目录**

用法和 `os.CreateTemp` 非常相似，签名长这样：

```go
dir, err := os.MkdirTemp("", "myapp-*")
```

第一个参数是父目录，传空字符串就用系统默认的临时目录。第二个参数是名字前缀（含 pattern）。返回值是新建好的临时目录**完整路径字符串**，不是文件句柄——因为目录本身不需要读写句柄，直接用路径操作就好。

---

**什么时候用临时目录而不是临时文件？**

如果你只需要一个文件，`CreateTemp` 够了。但现实里经常遇到这种情况：要生成一批中间文件，比如解压一个压缩包、跑一次代码生成、处理一组上传图片……这时候把它们全塞进同一个临时目录，逻辑清晰，而且清理超简单——一句 `os.RemoveAll` 就把整棵目录树连根拔掉，一个不留。

---

**defer + RemoveAll，函数结束自动打扫**

标准写法是这样：

```go
dir, err := os.MkdirTemp("", "upload-*")
if err != nil {
    log.Fatal(err)
}
defer os.RemoveAll(dir) // 函数退出时自动清光
```

`defer` 保证不管函数正常返回还是中途报错，清理动作都会执行。这是 Go 里处理临时资源的经典搭档，写成习惯就不会忘。

---

**在临时目录里创建文件：用 filepath.Join 拼路径**

拿到临时目录路径之后，用 `filepath.Join` 安全拼接文件名：

```go
fpath := filepath.Join(dir, "result.json")
err = os.WriteFile(fpath, data, 0600)
```

不要手动用字符串拼接路径，`filepath.Join` 会自动处理分隔符，跨平台也没问题。

---

**pattern 里的星号 * 到底放哪里？**

这一点和 `CreateTemp` 一样规则：pattern 字符串里如果包含 `*`，随机字符串会替换 `*` 所在位置；如果不含 `*`，随机字符串直接追加在末尾。举个例子：

- `"cache-*-tmp"` → 生成类似 `cache-839571204-tmp` 的目录名
- `"cache"` → 生成类似 `cache839571204` 的目录名

前者让你可以把随机段夹在有意义的前后缀之间，命名更灵活。

---

**安全提醒：千万别用可预测的名字**

有的同学图省事，直接用 `os.Mkdir("/tmp/myapp", 0700)` 这种固定名字。这在多用户环境下很危险——攻击者可以提前在那个路径建一个软链接，指向系统敏感目录，你的程序往里写数据，实际上写到了别的地方，造成**符号链接攻击（symlink attack）**。使用 `MkdirTemp` 生成带随机串的目录名，攻击者无法预测路径，这个风险就消除了。

---

**实战场景快速盘点**

第一，**测试夹具**：在 Go 测试里，`t.TempDir()` 一行搞定，框架自动在测试结束后清理，不用手写 `defer`，推荐优先用这个。

第二，**构建缓存**：编译器、代码生成工具把中间产物写进临时目录，成功后再移到目标位置，失败了也不会留下垃圾。

第三，**临时上传区**：用户上传文件先落到临时目录，校验、病毒扫描通过后再正式入库，整个过程干净可控。

掌握这几个组合拳，临时文件和目录就完全不是问题了。

## 🏃 跑一下试试

跑 `go run temporary-files-and-directories.go`，你会看到两行输出，类似：`Temp file name: /tmp/sample3312345` 和 `Temp dir name: /tmp/sampledir445566`。每次运行数字后缀都不同，这是 os 包在内部用随机字符串保证唯一性，避免并发场景下多个进程抢占同一文件名。两个 defer 语句（os.Remove 和 os.RemoveAll）在 main 函数返回时自动触发，操作系统层面彻底清理，不会留下垃圾文件。整个流程：创建 → 使用 → 自动删除，干净利落。

## 💡 师兄的碎碎念

💡 **pattern 里 * 的位置可以自定义**："sample*.log" 会生成 "sample839201.log"，把随机串夹在中间，方便按扩展名归类。

💡 **写测试时优先用 t.TempDir()**：Go 1.15 起 testing.T 提供 TempDir 方法，测试结束自动清理，比手写 defer RemoveAll 更简洁且不怕 panic 导致泄漏。

💡 **并发安全靠随机后缀**：os.CreateTemp 内部使用加密级随机源生成后缀，多个 goroutine 或进程同时调用也不会碰撞。

💡 **ioutil 已退休**：Go 1.16 之前用 ioutil.TempFile / ioutil.TempDir，现在统一迁移到 os.CreateTemp / os.MkdirTemp，旧代码看到 ioutil 直接替换即可。

💡 **Windows 路径不同**：第一个参数传空字符串时，Windows 使用 GetTempPath 返回的目录（通常是 %USERPROFILE%\AppData\Local\Temp），代码无需修改即可跨平台。

## 🎓 知识点清单

- os.CreateTemp("", "sample") 创建临时文件，返回 *os.File
- os.MkdirTemp("", "sampledir") 创建临时目录，返回路径字符串
- defer os.Remove(f.Name()) 确保临时文件在函数结束后自动删除
- defer os.RemoveAll(dname) 递归删除临时目录及其中所有内容
- f.Name() 获取操作系统分配的完整绝对路径
- 第一个参数传空字符串表示使用系统默认临时目录（Linux/macOS 为 /tmp）
- pattern 参数中的 * 标记随机字符串插入位置，不含 * 则追加在末尾
- 向临时文件写入数据：f.Write([]byte("data")) 与普通文件操作完全一致
