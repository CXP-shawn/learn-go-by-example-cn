## 🤔 先想一个问题

你有没有想过，为什么几乎所有编程语言都有"函数"这个概念？

想象你要写一个程序，计算很多组数字的和。如果没有函数，你每次都要重复写 `a + b`。函数的核心价值就是**封装可复用的逻辑**，让你一次定义，到处使用。

Go 的函数语法简洁而严格。今天我们就通过两个简单的加法函数，把 Go 函数的基础语法吃透。

---

## 📖 看代码

```go
// _函数_ 是 Go 的核心。我们将通过一些不同的例子来进行学习它。

package main

import "fmt"

// 这里是一个函数，接受两个 `int` 并且以 `int` 返回它们的和
func plus(a int, b int) int {

	// Go 需要明确的 return，也就是说，它不会自动 return 最后一个表达式的值
	return a + b
}

// 当多个连续的参数为同样类型时，
// 可以仅声明最后一个参数的类型，忽略之前相同类型参数的类型声明。
func plusPlus(a, b, c int) int {
	return a + b + c
}

func main() {

	// 通过 `函数名(参数列表)` 来调用函数，
	res := plus(1, 2)
	fmt.Println("1+2 =", res)

	res = plusPlus(1, 2, 3)
	fmt.Println("1+2+3 =", res)
}
```

## 🔍 师兄给你逐行拆

好，代码看起来很简单，但每个细节都值得好好说说。师兄带你逐行拆解。

### 函数定义语法

Go 定义函数的语法是：

```
func 函数名(参数列表) 返回值类型 {
    函数体
}
```

让我们看第一个函数：

```go
func plus(a int, b int) int {
    return a + b
}
```

**逐部分拆解：**

- `func`：这是 Go 的关键字，用来声明一个函数。任何函数定义都以 `func` 开头。
- `plus`：函数名。Go 遵循驼峰命名法，首字母小写表示包内私有，首字母大写表示可被外部包访问。这里 `plus` 小写，只在当前包内用。
- `(a int, b int)`：参数列表。每个参数的格式是"变量名 类型"，多个参数用逗号分隔。这和很多语言（如 Java、C）把类型写在变量名前面不同，Go 把类型写在变量名**后面**。
- `int`（括号外面的那个）：返回值类型。Go 的返回值类型写在参数列表之后，函数体之前。这也是 Go 语法的特点之一。
- `return a + b`：显式返回语句。

### 为什么 Go 要求显式 return？

注释里说了一句很重要的话：

> Go 需要明确的 return，也就是说，它不会自动 return 最后一个表达式的值

这和 Ruby、Elixir 等语言不同。在那些语言里，函数会自动把最后一个表达式的值作为返回值。但 Go 不会这样。你必须明确写 `return`。

这样做的好处是：代码意图更清晰，阅读者一眼就知道函数在哪里返回了什么值，减少歧义。

如果你定义了返回值类型但函数体里没有 `return`，Go 编译器会直接报错。

### 多个相同类型参数的简写

看第二个函数：

```go
func plusPlus(a, b, c int) int {
    return a + b + c
}
```

这里参数列表是 `a, b, c int`，而不是 `a int, b int, c int`。

当连续多个参数的类型相同时，Go 允许你只在最后一个参数后面写类型，前面的参数名之间用逗号分隔。这是一种简写语法，让参数列表更简洁。

等价写法对比：
```go
// 完整写法
func plusPlus(a int, b int, c int) int { ... }

// 简写（等价）
func plusPlus(a, b, c int) int { ... }
```

两种写法完全等效。在参数比较多时，简写可以让函数签名更加清爽。

### 函数调用

```go
res := plus(1, 2)
fmt.Println("1+2 =", res)

res = plusPlus(1, 2, 3)
fmt.Println("1+2+3 =", res)
```

调用函数的语法是 `函数名(参数1, 参数2, ...)`。

第一行 `res := plus(1, 2)`：用短变量声明 `:=` 创建新变量 `res`，并把 `plus(1, 2)` 的返回值（3）赋给它。

第二行 `res = plusPlus(1, 2, 3)`：注意这里是 `=` 而不是 `:=`，因为 `res` 已经声明过了，这里只是重新赋值。

这个细节值得记住：`:=` 用于**声明并赋值**，`=` 用于**仅赋值**（变量必须已存在）。

### 函数的位置

你可能注意到，`plus` 和 `plusPlus` 函数定义在 `main` 函数之前。在 Go 中，函数的定义顺序不重要，`main` 函数可以调用在它之后定义的函数，编译器在编译时会把整个包的所有函数都找到。不像 C 语言需要在调用前先声明函数原型。

### 返回值的类型安全

Go 是强类型语言。`plus` 函数返回 `int`，所以 `res` 的类型就是 `int`。你不能把一个返回 `string` 的函数的返回值赋给一个 `int` 类型的变量，编译器会报错。这种严格的类型系统帮助我们在编译阶段就发现很多错误。

### 函数是"一等公民"

在 Go 中，函数是一等公民（First-class citizen），这意味着函数可以像普通值一样被赋给变量、作为参数传递、作为返回值返回。这个特性让 Go 能够支持函数式编程风格，非常强大。虽然这个例子里还没涉及到，但学到后面的闭包、匿名函数时你会深刻体会到。

### 小结：Go 函数的核心规则

总结一下 Go 函数最基础的规则：
1. 用 `func` 关键字定义函数
2. 参数格式是"变量名 类型"，类型写在变量名后面
3. 返回值类型写在参数列表后面
4. 函数体里必须有 `return` 语句（有返回值时）
5. 相同类型的连续参数可以共用一个类型声明
6. 函数调用格式：函数名加括号，括号里放参数

## 💡 师兄的碎碎念

**Go 函数名的大小写决定可见性**

在 Go 中，函数名的首字母大小写有特殊含义：
- **大写开头**（如 `Println`）：导出函数（Exported），可以被其他包调用
- **小写开头**（如 `plus`）：未导出函数（Unexported），只能在当前包内使用

这是 Go 的可见性规则，简单优雅，不需要 `public`、`private` 关键字。

**Go 没有函数重载**

很多语言（如 Java）支持函数重载，即同名函数可以有不同的参数列表。Go **不支持**函数重载。每个函数名在同一个包内必须唯一。如果你需要类似功能，可以用不同的函数名（如 `plus` 和 `plusPlus`），或者使用可变参数（下一关会讲到）。
