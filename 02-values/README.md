# 第 2 关：Values（师兄带你学 Go）

## 🎯 这一关你会学到

这一关你会认识 Go 语言里最基础的几种

## 🤔 先想一个问题

想象你在奶茶店点单。店员问你："要不要加糖？要不要加冰？" 你回答"要" 或 "不要" ——这就是 **布尔值**（true/false）；你点了一杯 38 元的奶茶，38 这个数字就是 **整数**；如果打了九折是 34.2 元，那个小数就是 **浮点数**；而你对朋友说的 "大杯椰果少冰" 这串文字，就是 **字符串**。编程语言需要区分这些不同

## 📖 看代码

```go
// Go 拥有多种值类型，包括字符串、整型、浮点型、布尔型等。
// 下面是一些基础的例子。

package main

import "fmt"

func main() {

 // 字符串可以通过 `+` 连接。
 fmt.Println("go" + "lang")

 // 整数和浮点数
 fmt.Println("1+1 =", 1+1)
 fmt.Println("7.0/3.0 =", 7.0/3.0)

 // 布尔型，以及常见的布尔操作。
 fmt.Println(true && false)
 fmt.Println(true || false)
 fmt.Println(!true)
}
```

## 🔍 师兄给你逐行拆

### 第一段：字符串拼接

```go
fmt.Println("go" + "lang")
```

这行代码把两个字符串 `"go"` 和 `"lang"` 用 `+` 号拼在一起，输出结果是 `golang`。

生活类比：就像把两张便利贴撕下来粘在一起，"早上好" + "同学" = "早上好同学"。在 Go 里，`+` 对字符串的含义是**拼接**，而不是加法。这里要注意一个容易踩的坑：**`+` 只能拼同类型的东西**。你不能写 `"价格：" + 38`，因为一个是字符串、一个是整数，Go 编译器会直接报错，不像某些语言会偷偷帮你转换。Go 在类型这件事上非常严格，这其实是好事，能在编译阶段帮你揪出很多低级 bug。

---

### 第二段：整数与浮点数运算

```go
fmt.Println("1+1 =", 1+1)
fmt.Println("7.0/3.0 =", 7.0/3.0)
```

第一行：`fmt.Println` 可以同时接收多个参数，用逗号隔开，输出时它会自动在中间加一个空格。所以输出是 `1+1 = 2`，不是 `1+1 =2`。这个细节很多新手会忽略。

第二行：`7.0/3.0` 的结果是 `2.3333333333333335`，注意这里写的是 `7.0` 而不是 `7`。**这个区别至关重要！** 在 Go 里，`7/3` 是整数除法，结果是 `2`（直接截断小数部分，不四舍五入）；而 `7.0/3.0` 是浮点数除法，才会得到带小数的结果。这就好比你在宿舍分外卖，6 个鸡翅 4 个人分，整数除法告诉你每人分 1 个，剩下的 2 个不知道去哪了——而浮点数除法才会告诉你每人理论上能分到 1.5 个。实际写代码时，除法的类型错误是一个非常高频的 bug 来源，务必留意你的数字后面有没有小数点。

---

### 第三段：布尔值与逻辑运算

```go
fmt.Println(true && false)
fmt.Println(true || false)
fmt.Println(!true)
```

布尔值只有两个：`true`（真）和 `false`（假），是 Go 的关键字，**必须小写**，写成 `True` 或 `TRUE` 都会报错，这点和 Python 不一样。

三个运算符逐一拆解：

- `&&`（与）：两个条件**都要为真**，结果才是 true。类比：进宿舍楼，你**既要**刷校园卡**又要**人脸识别，缺一个都进不去。`true && false` = `false`。
- `||`（或）：两个条件**有一个为真**，结果就是 true。类比：买奶茶，**要么**微信支付**要么**支付宝，能付就行。`true || false` = `true`。
- `!`（非）：取反，true 变 false，false 变 true。类比：今天**不**下雨，就出去打球。`!true` = `false`。

这三个运算符是所有条件判断、流程控制的基础，后面写 `if` 语句时你会大量用到它们，现在先把符号和语义对应上。

---

### 总结一下这关的隐藏知识点

Go 是**静态强类型语言**，每一个值都有明确的类型，不同类型之间不能随便混用。这在初学时会觉得有点烦，但随着项目变大你会越来越感激它——编译器帮你把类型问题拦在门外，比运行时崩溃好太多了。

## 💡 师兄的碎碎念

- `7.0/3.0` 结果末尾是 `...335` 不是 `...333`，别以为是 Bug——这是 IEEE 754 浮点精度问题，所有语言都这样，Go 只是诚实地告诉你而已。
- 试着把 `7.0/3.0` 改成 `7/3`，你会发现输出变成 `2`，整数除法直接砍掉小数部分，比你期末挂科还无情。
- Go 的 `bool` 类型只有 `true` 和 `false`，不像 Python/JS 那样 `0`、空字符串也能当 false 用，想走捷径的同学要改改习惯。
- `fmt.Println` 可以同时传多个参数，参数之间自动加空格，所以 `"1+1 =", 1+1` 输出是 `1+1 = 2` 中间有空格，这个小细节面试偶尔会考。

## 🎓 这一关的知识点清单

- **基本值类型**：Go 内置 `string`、`int`、`float64`、`bool` 等类型，类型在编译期就确定，不像 JS 那样随便变身。
- **字符串拼接**：用 `+` 连接字符串，简单粗暴；但大量拼接时推荐用 `strings.Builder`，否则内存会哭。
- **整数与浮点数区别**：`1+1` 是整数运算，`7.0/3.0` 是浮点运算，Go 不会自动帮你升级类型，整数除整数永远是整数，别指望 `7/3` 等于 `2.333`。
- **布尔运算**：`&&`（与）、`||`（或）、`!`（非）和数学逻辑一样，`&&` 短路求值，左边假了右边直接跳过。
- **强类型语言**：Go 不允许不同类型直接运算，`1 + 1.5` 会编译报错，必须显式转换，这是新手踩的第一个大坑。
