# 第 5 关：For（师兄带你学 Go）

## 🎯 这一关你会学到

这一关你会掌握 Go 语言里**唯一的循环结构** `for` 的三种写法：像 while 一样只写条件、经典三段式、以及无限循环配合 `break`/`continue` 控制流程。Go 把 `while`、`do-while`、`for` 全部合并成一个 `for` 关键字，化繁为简，学一个顶仨。

## 🤔 先想一个问题

想象你在奶茶店打工，店长说：「把今天到货的珍珠一颗一颗数，数够 500 颗就收工。」你不可能站在那里喊 500 次「数一颗」，你肯定会**重复**同一个动作，直到满足某个条件为止。这种「重复执行，直到达到目标」的模式，在编程里就叫**循环（Loop）**。

Go 语言的设计者极度信奉「少即是多」，别的语言给你 `while`、`do-while`、`for` 三种循环任你挑，Go 直接说：不，一个 `for` 就够了，其他都是花里胡哨。`for` 是 Go 中**唯一的循环关键字**，但它能伪装成多种形态，灵活度完全不输那些

## 📖 看代码

```go
// `for` 是 Go 中唯一的循环结构。这里会展示 `for` 循环的一些基本使用方式。

package main

import "fmt"

func main() {

 // 最基础的方式，单个循环条件。
 i := 1
 for i <= 3 {
 fmt.Println(i)
 i = i + 1
 }

 // 经典的初始/条件/后续 `for` 循环。
 for j := 7; j <= 9; j++ {
 fmt.Println(j)
 }

 // 不带条件的 `for` 循环将一直重复执行，
 // 直到在循环体内使用了 `break` 或者 `return` 跳出循环。
 for {
 fmt.Println("loop")
 break
 }

 // 你也可以使用 `continue` 直接进入下一次循环。
 for n := 0; n <= 5; n++ {
 if n%2 == 0 {
 continue
 }
 fmt.Println(n)
 }
}
```

## 🔍 师兄给你逐行拆

### 第一段：只写条件的 `for`（Go 版 while）

```go
i := 1
for i <= 3 {
    fmt.Println(i)
    i = i + 1
}
```

这就是那个「数珍珠」的奶茶店工人版本。`i` 从 1 开始，每次数完一颗就 `+1`，条件是「还没超过 3」就继续。`for i <= 3` 翻译成人话就是：**「只要 i 还小于等于 3，就一直干」**。

这种写法和其他语言的 `while (i <= 3)` 完全等价。Go 没有 `while` 关键字，但这里的 `for` 就是它。

⚠️ **最容易踩的坑**：如果你忘了写 `i = i + 1`，`i` 永远是 1，条件永远成立，程序就会无限打印下去——这叫**死循环**，就像奶茶店工人发现珍珠桶是无底洞，永远数不完，电脑只能强制重启（Ctrl+C）解救。

---

### 第二段：经典三段式 `for`

```go
for j := 7; j <= 9; j++ {
    fmt.Println(j)
}
```

这是最「教科书」的写法，三段用分号隔开：
- **初始化**：`j := 7`，进循环前只执行一次，相当于「把计数器拨到 7」。
- **条件判断**：`j <= 9`，每轮开始前检查，不满足就立刻退出。
- **后续操作**：`j++`，每轮结束后自动执行，等价于 `j = j + 1`。

生活类比：就像游戏里的闯关进度条，从第 7 关开始（初始化），每关打完自动推进（j++），打完第 9 关就结束（条件不满足退出）。

注意 `j` 的**作用域**只在 `for` 循环内部，出了花括号就访问不到了。这是 Go 的设计——变量关在它该待的地方，省得到处乱跑。

---

### 第三段：无限循环 + `break`

```go
for {
    fmt.Println("loop")
    break
}
```

`for` 后面什么都不写，就是**无限循环**，相当于「一直干，没有停下来的理由」。但聪明的你马上发现——那不就死循环了吗？对，所以要在循环体内用 `break` 手动踩刹车。

这种写法常见于**服务器监听**场景：一个 Web 服务器需要一直等待用户请求，没有请求就等着，有请求就处理，永远不主动退出——直到程序被人为关掉。`break` 则是在满足某个特定条件时（比如收到「关机信号」）才触发。

---

### 第四段：`continue` 跳过本轮

```go
for n := 0; n <= 5; n++ {
    if n%2 == 0 {
        continue
    }
    fmt.Println(n)
}
```

`n%2 == 0` 判断 n 是不是偶数（能被 2 整除余 0）。如果是偶数，`continue` 直接跳到下一轮，`fmt.Println(n)` 这行**不会执行**。最终只打印奇数：1、3、5。

`continue` 和 `break` 的区别：
- `break` = 整个循环我不干了，走人
- `continue` = 这一轮我跳过，但下一轮我还来

就像你在食堂打饭，遇到不喜欢的菜就跳过（continue），但不是直接甩盘子走人（break）。

---

总结一下：Go 的 `for` 像个变形金刚，三段式、单条件、无限循环三种形态随需切换，配合 `break` 和 `continue` 能处理绝大多数循环场景。记住这一个关键字，抵得上别的语言学三个！

## 💡 师兄的碎碎念

- Go 里根本没有 `while` 关键字，所有循环统一用 `for` 扛，这是 Go 设计者「少即是多」哲学的典型体现——就像食堂只卖一种套餐，你别无选择，但其实挺香的。
- 三段式 `for j := 7; j <= 9; j++` 里，`j` 的作用域仅限于这个 for 循环内部，出了大括号就消失了，别想着在外面用它，会编译报错，亲测翻车。
- `for {}` 无限循环一定要有退出条件，漏写 `break` 或 `return` 的话程序会把你的 CPU 当免费劳动力，风扇狂转、电脑发烫，室友会来问你在挖矿吗。
- `continue` 只影响「当前这一轮」，不是退出整个循环，很多新手把它和 `break` 搞混，记住：`continue` 是"跳题", `break` 是"交卷走人"，完全不同的概念。

## 🎓 这一关的知识点清单

- **单条件 for**：`for 条件 {}` 等价于其他语言的 `while`，Go 直接把 `while` 给开除了，只留 `for` 一个打工人干所有活。
- **三段式 for**：`for 初始化; 条件; 后续 {}` 是最经典的 C 风格写法，初始化变量、判断条件、每轮结束后执行后续，三步走缺一不可。
- **无限循环 for**：`for {}` 不带任何条件，会永远跑下去，必须靠 `break` 或 `return` 手动踩刹车，不然程序就成老实人了——一直干活不停歇。
- **continue 跳过**：`continue` 不是退出循环，而是跳过本轮剩余代码直接进入下一轮，相当于"这道题不会，跳过，下一题"。
- **break 终止**：`break` 是直接从当前循环逃跑，后面的轮次全不执行了，是真正意义上的"老子不干了"。
